# Internationalization (i18n) Guide

> Last updated: 2026-06-30

| Code    | Language | Status                 |
| ------- | -------- | ---------------------- |
| `en`    | English  | ✅ Complete (fallback) |
| `zh-CN` | 简体中文 | ✅ Complete            |
| `zh-TW` | 繁體中文 | ✅ Complete            |

## How It Works

### Language Detection

The app uses `i18next-browser-languagedetector` with this priority order:

1. **Query string** — `?lng=zh-CN`
2. **localStorage** — `i18nextLng` key
3. **Browser language** — `navigator.language`
4. **Fallback** — `en`

Users can manually switch via the globe icon in the header. Their choice is persisted to `localStorage`.

### File Structure

```
packages/app/src/i18n/
├── index.ts              # i18next init + language list
└── locales/
```

## Adding a New Language

1. **Create locale file**: `src/i18n/locales/xx.json`
   - Copy `en.json` as template
   - Translate all values

2. **Register in i18n config**: `src/i18n/index.ts`

   ```ts
   import xx from './locales/xx.json';

   // In resources:
   xx: { translation: xx },
   ```

3. **Add to language list**: `src/i18n/index.ts`

   ```ts
   { code: 'xx', label: 'Language Name' }
   ```

4. **Verify**: Run dev server, switch language in header dropdown
5. **Check parity**: compare flattened keys against `en.json`
6. **Check fonts**: confirm text renders without missing glyphs or awkward fallback metrics

## Adding New Translation Keys

1. Add the key to `en.json` first
2. Add translations to all other locale files
3. Use in component:

   ```tsx
   import { useTranslation } from 'react-i18next';

   function MyComponent() {
     const { t } = useTranslation();
     return <Text>{t('namespace.key')}</Text>;
   }
   ```

## Pluralization

English uses `_one` / `_other` suffixes:

```json
{
  "fileQueue": {
    "title_one": "{{count}} file queued",
    "title_other": "{{count}} files queued"
  }
}
```

Chinese/Japanese/Korean still include both suffixes in this project so every locale has the same key shape:

```json
{
  "fileQueue": {
    "title_one": "{{count}} 个文件已添加",
    "title_other": "{{count}} 个文件已添加"
  }
}
```

Usage: `t('fileQueue.title', { count: 5 })`

## Interpolation

Dynamic values use `{{variable}}` syntax:

```json
{ "downloadZip": "Download ZIP ({{count}})" }
```

Usage: `t('actions.downloadZip', { count: 10 })`

## Technical Terms

Format names (MozJPEG, WebP, AVIF, OxiPNG) are kept as-is across all languages since they are industry-standard technical terms.
